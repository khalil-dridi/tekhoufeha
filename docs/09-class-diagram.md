# Design Class Diagram

## 1. Purpose

The design class diagram represents the software structure of the TekhouFeha platform. Unlike the analysis class diagram, this model includes implementation details such as attributes, operations, data types, enumerations, and class relationships. It serves as the blueprint for the backend implementation using Spring Boot and JPA.

## 3. Main Classes

The system is composed of the following main classes:

- User
- Product
- Category
- Offer
- Conversation
- Message
- Notification
- Report

These classes represent the core business entities of the TekhouFeha marketplace.

---

## 4. Enumerations

Several enumerations are used to improve readability, maintainability, and type safety.

| Enumeration | Description |
|-------------|-------------|
| Role | Defines the role of a user (ADMIN, USER). |
| UserStatus | Represents the current status of a user account. |
| SellingMode | Defines whether a product is sold at a fixed price or through negotiation. |
| ProductCondition | Indicates the physical condition of a product. |
| ProductStatus | Represents the availability status of a product. |
| OfferStatus | Represents the state of an offer during the negotiation process. |
| NotificationType | Specifies the type of notification generated by the system. |
| NotificationStatus | Indicates whether a notification has been read. |
| ReportStatus | Represents the processing status of a report. |

---

## 5. Relationships

The design class diagram defines the following relationships:

- A **User** can publish multiple **Products**.
- A **Product** belongs to one **Category**.
- A **Product** can receive multiple **Offers**.
- An accepted **Offer** may create one **Conversation**.
- A **Conversation** contains one or more **Messages**.
- A **User** can receive multiple **Notifications**.
- A **User** can submit multiple **Reports**.

---

## 6. Design Decisions

The following design decisions were adopted:

- UUID is used as the identifier for all business entities.
- Enumerations are used instead of String values for statuses and roles.
- Date and time information is represented using LocalDateTime.
- Each entity contains only the attributes and operations related to its own responsibility.
- The design is compatible with Spring Boot, Spring Data JPA, and a future microservices architecture.

---

## 7. Notes

This design class diagram represents the MVP version of TekhouFeha. Additional entities such as ProductImage, Favorite, CounterOffer, RefreshToken, Attachment, and ProductView may be introduced in future iterations as the platform evolves.
